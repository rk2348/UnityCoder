第3回: 変更の保存 (Commit & Push)

公開日: 2025.12.06

1. Unityで変更を加える

それでは、実際にUnityプロジェクトに変更を加えて、Gitで記録してみましょう。
Unityエディタを開いて、リポジトリに登録したプロジェクトで以下の作業を行ってください。

  • Hierarchy で右クリックし、3D Object > Cube を作成する。
  • Project ウィンドウで Scripts フォルダを作成し、その中に適当なC#スクリプトを作成する。
  • Ctrl + S (Macなら Cmd + S) を押して、シーンを保存する。

保存が終わったら、Unityを最小化して GitHub Desktop の画面に戻ります。

2. 変更内容を確認する

GitHub Desktopの左側にあるパネル（Changesタブ）を見てください。
今、Unityで行った変更が自動的に検出され、リストアップされているはずです。

【画像: Changesリストに .unity ファイルや .cs ファイルなどが並んでいる画面】

ファイルの左側にあるアイコンは、変更の種類を表しています。

  • [ + ] 緑色： 新規追加されたファイル
  • [ ● ] 黄色： 内容が変更されたファイル
  • [ - ] 赤色： 削除されたファイル

3. コミット (Commit) する

これらの変更を「歴史の1ページ」として確定・保存することを「コミット (Commit)」と呼びます。
ゲームで言えば「セーブデータを1つ作る」ような感覚です。

手順

  1. 画面左下の入力欄 Summary (Required) に、今回の変更内容を簡潔に入力します。
    例：「Cubeを追加」「プレイヤースクリプトを作成」など。
  2. 詳細が必要な場合は Description に追記しますが、空欄でも構いません。
  3. 青いボタン Commit to main をクリックします。
【画像: Summaryを入力してCommitボタンを押す直前の状態】
重要：
コミットボタンを押すとChangesリストが空になりますが、消えたわけではありません。
「記録済み」の状態になっただけです。これで手元のパソコン内に履歴が保存されました。

4. プッシュ (Push) する

コミットしただけでは、まだデータはあなたのパソコンの中にしかありません。
これをインターネット上のGitHubサーバーにアップロードしてバックアップすることを「プッシュ (Push)」と呼びます。

手順

  1. 画面の右上（または中央）にある Publish branch ボタンをクリックします。
    （※2回目以降は「Push origin」という表示に変わります）
  2. アップロードが完了するまで少し待ちます。
【画像: 右上の Push origin (または Publish branch) ボタン】

これで作業は完了です！
もしパソコンが壊れても、GitHubからプロジェクトを復元できるようになりました。
