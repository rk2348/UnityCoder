第4回: 履歴の確認と取り消し

公開日: 2025.12.06

1. 履歴（History）を見る

「いつ」「誰が」「どのような変更をしたか」を確認するには、GitHub Desktopの左側にある History タブをクリックします。
これまでのコミット履歴が時系列順に表示されます。

【画像: Historyタブをクリックし、過去のコミットログが一覧表示されている画面】

各コミットをクリックすると、右側のパネルにその時の変更内容（どのファイルのどの行が変わったか）が表示されます。
これにより、「昨日の作業でどこをいじったっけ？」という確認が簡単にできます。

2. まだコミットしていない変更を取り消す (Discard)

Unityで作業中に、「コードを書き換えていたらエラーだらけになってしまった！」「やっぱり今の変更は全部なかったことにして、最後のセーブ時点に戻したい」と思うことがあります。
Gitを使っていれば、コミット前の変更は一瞬で元に戻せます。

手順

  1. Changes タブ（作業中の変更一覧）を開きます。
  2. 元に戻したいファイルを右クリックします。
  3. Discard changes... を選択します。
  4. 確認ダイアログが出るので、赤いボタン（Discard）を押します。
【画像: Changesリストでファイルを右クリックし、Discard changesを選択している画面】
注意：
Discardを実行すると、そのファイルは「最後にコミットした状態」に上書きされます。
一度Discardすると、消した内容を復元することは絶対にできません。慎重に行ってください。

3. すでにコミットした変更を取り消す (Revert)

「コミットしてしまった後でバグに気づいた！」という場合もあります。
Gitでは「時間を巻き戻す」のではなく、「そのコミットの変更を打ち消すような新しいコミットを作る」ことで元に戻します。これを Revert（リバート） と呼びます。

手順

  1. History タブを開きます。
  2. 取り消したい過去のコミットを右クリックします。
  3. Revert changes in commit を選択します。
【画像: Historyリストで右クリックし、Revert changes in commitを選択している画面】

これを実行すると、「Revert "コミット名"」という新しいコミットが自動的に作成されます。
ファイルの中身は元に戻りますが、「一度間違えて、それを修正した」という履歴自体は残るため、チーム開発でも安全に過去の状態に戻すことができます。

完了：
Revertを行った後は、忘れずに Push origin を押して、取り消した内容をGitHubにも反映させましょう。
