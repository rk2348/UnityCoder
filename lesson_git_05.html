第5回: ブランチとマージ

公開日: 2025.12.06

1. ブランチ（Branch）とは？

「ブランチ」とは、履歴を木の枝のように分岐させて、メインのプロジェクトに影響を与えずに別の作業をする機能です。
例えば、「新しい敵AIを作りたいけれど、失敗してゲーム全体が動かなくなるのは怖い」という場合に、実験用の「枝（ブランチ）」を作ってそこで作業します。

【画像: ブランチの概念図（メインの線から別の線が枝分かれしている図）】

初期状態では main という一本のブランチにいますが、ここから自由に分岐させることができます。

2. ブランチを作成する

実際に新しいブランチを作ってみましょう。

手順

  1. GitHub Desktop画面上部にある Current Branch をクリックします。
  2. New Branch ボタンをクリックします。
  3. ブランチ名を入力します（例: feature-enemy-ai）。
    ※英語で、スペースの代わりにハイフンを使うのが一般的です。
  4. Create Branch をクリックします。
【画像: New Branch作成ウィンドウで名前を入力している画面】

これで、あなたの作業場所は feature-enemy-ai ブランチに切り替わりました。
この状態でUnityでファイルを変更しても、main ブランチには一切影響しません。安心して実験的なコードを書くことができます。

3. ブランチを切り替える

ブランチの切り替えは一瞬でできます。

  • 画面上部の Current Branch から main を選ぶと、ファイルの中身が「分岐する前の状態」にパッと戻ります。
  • また feature-enemy-ai を選ぶと、実験中の状態に戻ります。
注意：
Unityエディタを開いたままブランチを切り替えると、Unityが「Reload」を行います。シーン保存していない変更は消える可能性があるので、必ずCtrl+Sで保存してから切り替えてください。

4. マージ（Merge）で統合する

実験用のブランチで機能が完成し、バグがないことも確認できました。
この変更をメイン（main）に取り込む作業を「マージ (Merge)」と呼びます。

手順

  1. まず、取り込まれる側（main）にブランチを切り替えます。
    （※ここを間違えやすいので注意！「main」に移動してください）
  2. メニューバーの Branch > Merge into current branch... を選択します。
  3. リストから、取り込みたいブランチ（feature-enemy-ai）を選択します。
  4. 下の青いボタン Create a merge commit (または Merge feature-...) をクリックします。
【画像: Merge into current branchのダイアログ画面】

これで、実験用ブランチの内容が main ブランチに合体しました！
不要になった実験用ブランチは削除しても構いません。
