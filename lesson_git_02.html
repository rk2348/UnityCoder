第2回: リポジトリ作成と設定

公開日: 2025.12.06

1. リポジトリとは？

Gitで管理されているフォルダのことを「リポジトリ (Repository)」と呼びます。
今回は、あなたのパソコンにあるUnityプロジェクトをリポジトリとして登録する手順を解説します。

2. 新規リポジトリの作成

GitHub Desktopを使って、新しいリポジトリを作成します。

手順1: 作成画面を開く

  1. GitHub Desktopを起動します。
  2. 左上のメニューバーから [File] をクリックします。
  3. [New Repository...] を選択します。

※既存のフォルダを登録する場合でも、まずはこの「New Repository」から行うのが最も安全で簡単です。

【画像: GitHub Desktopの File > New Repository メニュー画面】

3. 最重要：Git Ignoreの設定

ここがUnity開発において最も重要なステップです。設定を間違えると、不要なファイルまでアップロードされてトラブルの原因になります。

手順2: リポジトリ情報の入力

表示されたウィンドウで、以下の項目を入力・選択してください。

  • Name: プロジェクト名を入力します（例: MyShootingGame）。
  • Local Path: プロジェクトを作成したい場所（親フォルダ）を選択します。
  • Git Ignore: プルダウンメニューから「Unity」を探して選択します。
  • License: 特になければ「None」のままでOKです。
【画像: New Repository画面でGit Ignoreに「Unity」を選択している状態】
なぜGit Ignoreが必要？
Unityプロジェクトには Library や Temp といったフォルダが含まれます。これらはUnityが起動するたびに自動生成される巨大なデータで、Gitで管理する必要がありません。
Git Ignoreに「Unity」を設定することで、これらの不要なファイルを自動的に除外（無視）してくれます。

手順3: 作成の実行

入力内容を確認し、右下の「Create Repository」ボタンをクリックします。
これで、指定した場所に新しいフォルダが作成され、Gitの管理が開始されました。

4. 既存のUnityプロジェクトを管理したい場合

「すでに作ってあるゲームをGit管理したい」という場合は、以下の手順で行います。

  1. 上記の手順と同様に Local Path で、既存プロジェクトがある場所の「一つ上の階層」を指定します。
  2. Name に、既存のプロジェクトフォルダと全く同じ名前を入力します。
  3. 「このフォルダは既に存在します」という警告が出ても、そのまま作成を進めます（Gitの管理ファイルだけが追加されます）。
準備完了！
これでリポジトリの作成は完了です。次回は、実際にUnityで変更を加えて、その履歴を保存（コミット）する方法を学びます。
