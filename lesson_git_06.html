第6回: コンフリクトの解消

公開日: 2025.12.06

1. コンフリクトとは？

チーム開発などで、AさんとBさんが「同じファイルの同じ行」を同時に変更してしまった場合、Gitはどちらの変更を採用すべきか自動的に判断できません。
この状態を「コンフリクト（競合）」と呼びます。

Unityユーザーへの重要事項：
C#スクリプト（テキストファイル）のコンフリクトは比較的簡単に直せますが、シーンファイル (.unity) や プレハブ (.prefab) でコンフリクトが起きると、中身が複雑すぎて手動での修復はほぼ不可能です。
チーム開発では「同じシーンを複数人で同時に触らない」「Prefabを活用して作業を分担する」といったルール作りが重要です。

2. コンフリクトの発生

GitHub Desktopで「Pull」や「Merge」を行った際、コンフリクトが発生すると以下のような警告画面が表示されます。

【画像: GitHub Desktopの「Conflicted files」という警告画面】

3. コンフリクトの解決手順 (スクリプトの場合)

警告画面が出ても慌てる必要はありません。以下の手順で解決します。

手順1: エディタで開く

  1. GitHub Desktopの警告画面にある Open in Visual Studio Code（またはお使いのエディタ）ボタンをクリックします。

手順2: 記号を探して修正する

ファイルを開くと、Gitが自動的に以下のような記号を挿入しています。

<<<<<<< HEAD
int hp = 100; // 自分の変更
=======
int hp = 200; // 相手の変更
>>>>>>> main

この <<<、===、>>> に囲まれた部分が競合箇所です。
どちらのコードが正しいか（あるいは両方必要か）を判断し、正しいコードだけを残して、記号の行はすべて削除してください。

手順3: マージコミットを作成する

  1. ファイルを保存して閉じます。
  2. GitHub Desktopに戻ると、ボタンが「Commit merge」に変わっているはずです。
  3. Commit merge をクリックして完了です。

4. 講座のまとめ

全6回の講座、お疲れ様でした！
これでGitを使った基本的な開発フロー（リポジトリ作成、コミット、プッシュ、ブランチ、マージ）は習得できました。

今後のアクション：
  • 自分のUnityプロジェクトをすべてGitHubにバックアップする。
  • 「一日一回は必ずコミット」を習慣にする。
  • 大きな機能追加をする時は、怖がらずにブランチを切ってみる。

バージョン管理を味方につけて、安全で快適なUnity開発ライフを送ってください！
